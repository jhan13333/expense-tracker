# 고정비 지출 트래커

매달 반복되는 고정비 항목을 관리하고, 각 항목에 대해 해당 월의 지출일과 지불 방법을 기록·조회할 수 있는 웹 애플리케이션입니다.

## 기능

### 주요 기능

- **항목 관리**
  - 고정비 항목 추가, 수정, 삭제
  - 항목명, 예상 금액, 비고 입력
  - 항목 활성화/비활성화

- **월별 지출 기록**
  - 월별 지출 상태 관리
  - 지출일, 지불 방법(신용카드, 계좌이체, 현금, 자동이체, 기타), 메모 기록
  - 지출 완료 여부 표시

- **필터 및 정렬**
  - 전체/미지출/지출완료 필터
  - 이름순/금액순/상태순 정렬
  - 항목명 검색

- **요약 정보**
  - 현재 선택 월의 지출 완료 개수 / 전체 항목 수
  - 완료율 표시
  - 완료 항목의 합계 금액

- **데이터 백업/복원**
  - JSON 형식으로 데이터 내보내기
  - JSON 파일 가져오기 (덮어쓰기 또는 병합)
  - 데이터 초기화

## 사용 방법

### 기본 사용

1. **항목 추가**
   - 상단의 "항목 추가" 섹션에서 항목명을 입력하고 (필수), 예상 금액과 비고를 입력할 수 있습니다 (선택).
   - "추가" 버튼을 클릭하면 목록에 항목이 추가됩니다.

2. **월별 지출 기록**
   - 상단의 월 선택기에서 관리할 월을 선택합니다.
   - 각 항목 카드에서 다음 정보를 입력할 수 있습니다:
     - 지출일: 해당 월의 날짜 범위 내에서 선택
     - 지불 방법: 신용카드, 계좌이체, 현금, 자동이체, 기타 중 선택
     - 메모: 추가 정보 입력 (선택)
     - 지출 완료: 체크박스로 완료 여부 표시

3. **항목 관리**
   - **수정**: 항목명, 금액, 비고를 수정할 수 있습니다.
   - **비활성화/활성화**: 항목을 일시적으로 숨기거나 다시 표시할 수 있습니다.
   - **삭제**: 항목을 삭제합니다 (확인 필요). 삭제된 항목의 지출 기록은 보존됩니다.

4. **필터 및 정렬**
   - 필터: 전체 항목, 미지출 항목, 지출 완료 항목만 보기
   - 정렬: 이름순, 금액순, 상태순으로 정렬
   - 검색: 항목명으로 검색

### 데이터 백업 및 복원

#### 데이터 내보내기

1. 하단의 "데이터 내보내기" 버튼을 클릭합니다.
2. JSON 파일이 다운로드됩니다.
3. 이 파일을 안전한 곳에 보관하세요.

#### 데이터 가져오기

1. 하단의 "데이터 가져오기" 버튼을 클릭합니다.
2. 이전에 내보낸 JSON 파일을 선택합니다.
3. 다음 중 하나를 선택합니다:
   - **덮어쓰기**: 기존 데이터를 모두 삭제하고 가져온 데이터로 교체
   - **병합**: 기존 데이터에 가져온 데이터를 추가 (중복 항목은 제외)

#### 데이터 초기화

- 하단의 "데이터 초기화" 버튼을 클릭하면 모든 데이터가 삭제됩니다.
- 이 작업은 되돌릴 수 없으므로 주의하세요.
- 데이터를 초기화하기 전에 반드시 백업을 받으세요.

## 기술 사양

- **기술 스택**: HTML5, CSS3, 바닐라 JavaScript (ES6+)
- **데이터 저장**: localStorage
- **브라우저 지원**: 최신 브라우저 (Chrome, Firefox, Safari, Edge)
- **반응형 디자인**: 모바일 우선 설계

## 데이터 구조

### 고정비 항목 (FixedExpense)

```javascript
{
  id: string,           // 고유 ID
  name: string,         // 항목명
  amount?: number,      // 월 예상 금액 (선택)
  note?: string,        // 비고 (선택)
  active: boolean,      // 사용 여부
  createdAt: string     // 생성일시 (ISO 형식)
}
```

### 월별 지출 기록 (PaymentRecord)

```javascript
{
  itemId: string,                    // 고정비 항목 ID
  yearMonth: string,                 // "YYYY-MM" 형식
  paidDate?: string,                 // 지출일 (ISO 형식)
  method?: string,                   // 지불 방법
  memo?: string,                     // 메모
  isPaid: boolean                    // 지출 완료 여부
}
```

## 주의사항

1. **지출일 유효성 검사**
   - 지출 완료를 체크할 때, 지출일은 반드시 선택한 월의 범위 내여야 합니다.
   - 예: 2025년 11월을 선택했다면, 지출일은 2025-11-01부터 2025-11-30 사이여야 합니다.

2. **중복 항목명**
   - 같은 이름의 항목은 추가할 수 없습니다 (대소문자 및 공백 무시).

3. **데이터 보존**
   - 항목을 삭제해도 해당 항목의 지출 기록은 보존됩니다.
   - 완전히 삭제하려면 데이터 초기화 후 필요한 항목만 다시 추가하세요.

4. **브라우저 호환성**
   - localStorage를 사용하므로, 브라우저의 쿠키/로컬 스토리지 설정에 따라 데이터가 삭제될 수 있습니다.
   - 정기적으로 데이터를 백업하는 것을 권장합니다.

## 문제 해결

### 데이터가 사라졌어요

- 브라우저의 쿠키/로컬 스토리지가 삭제되었을 수 있습니다.
- 정기적으로 데이터를 백업하세요.

### 지출일을 선택할 수 없어요

- 지출 완료를 체크하려면 먼저 지출일을 선택해야 합니다.
- 지출일은 선택한 월의 범위 내여야 합니다.

### 항목이 보이지 않아요

- 필터 설정을 확인하세요 (전체/미지출/지출완료).
- 검색어를 확인하세요.
- 항목이 비활성화되어 있을 수 있습니다.
